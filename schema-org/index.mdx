# Schema.org

**Schema.org** is a collaborative, community activity with a mission to create, maintain, and promote schemas for structured data on the Internet.

## Overview

Schema.org provides:
- **Type Hierarchy**: 800+ types organized in a taxonomy
- **Property Definitions**: 1,400+ properties describing type attributes
- **Vocabulary**: Shared understanding for structured data
- **Extensions**: Specialized vocabularies for specific domains
- **Integration**: Used by Google, Microsoft, Yahoo, Yandex for rich results

## Architecture

### Core Concepts

1. **Types**: Classes of things (Organization, Person, Product, etc.)
2. **Properties**: Attributes of types (name, description, url, etc.)
3. **Hierarchy**: Types extend other types via subClassOf
4. **Data Types**: Text, Number, Boolean, Date, URL, etc.

## Collections Generated

This source generates the following database collections:

### 1. Types (~800 entries)

**Path**: `db/Types/{slug}/readme.mdx`

Each type includes:
- Type name and URL
- Parent types (via rdfs:subClassOf)
- Description and comments
- Properties available on type
- Sub-types extending this type
- Usage examples

### 2. Properties (~1,400 entries)

**Path**: `db/Properties/{slug}/readme.mdx`

Each property includes:
- Property name and URL
- Domain (types that can have this property)
- Range (expected value types)
- Description and usage
- Examples

**Phase 2**: Extract properties to separate collection

## Relationships with Other Sources

### With O*NET (Occupations)

Schema.org **Occupation** type maps to O*NET:

```
schema:Occupation
  ↓
O*NET Occupation (15-1252.00: Software Developer)
  ├─> occupationalCategory: "15-1252.00"
  ├─> name: "Software Developer"
  └─> skills: [Programming, Systems Analysis]
```

### With NAICS (Industries)

Schema.org **Organization** maps to NAICS:

```
schema:Organization
  ├─> naics: "511210"
  ├─> industry: "Software Publishers"
  └─> isicV4: International equivalent
```

## Usage Examples

### Define an Organization

```typescript
import { Organization } from 'schema.org.ai'

const company: Organization = {
  '@type': 'Organization',
  name: 'Acme Corp',
  naics: '511210', // Software Publishers
  employees: {
    '@type': 'Occupation',
    occupationalCategory: '15-1252.00', // Software Developers
  }
}
```

### Query by Type

```typescript
import { db } from '@mastra/core'

const organizations = await db.types.findOne({
  where: { label: 'Organization' }
})

const subtypes = await db.types.findMany({
  where: { subClassOf: 'https://schema.org/Organization' }
})
```
