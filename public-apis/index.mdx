# Public APIs Directory

**Public APIs** is a collective list of free, public APIs maintained by the developer community on GitHub. It's one of the most popular and comprehensive API directories available, with over 250k GitHub stars.

## Overview

The Public APIs directory provides:
- **1400+ APIs**: Curated collection across diverse domains
- **40+ Categories**: Organized from Animals to Weather
- **Authentication Details**: Clear indication of auth requirements
- **HTTPS Support**: Security information for each API
- **CORS Status**: Cross-origin resource sharing compatibility
- **Community-Driven**: Actively maintained with contributions

## Data Structure

### API Entry Schema

Each API entry includes:

```typescript
interface PublicAPI {
  API: string                     // API name
  Description: string             // What the API provides
  Auth: string                    // Authentication type
  HTTPS: boolean                  // HTTPS support
  Cors: string                    // CORS support status
  Link: string                    // API documentation URL
  Category: string                // Primary category
}
```

### Authentication Types

- **No Auth**: Open access, no authentication required
- **apiKey**: Requires API key registration
- **OAuth**: OAuth 2.0 authentication
- **X-Mashape-Key**: RapidAPI authentication
- **User-Agent**: Requires User-Agent header

### CORS Status

- **yes**: Full CORS support
- **no**: No CORS support
- **unknown**: CORS status not verified

## Collections Generated

This source generates the following database collections:

### 1. Public APIs (~1400 entries)

**Path**: `db/API/{category}/{api-slug}/readme.mdx`

Each API includes:
- API name and description
- Documentation URL
- Authentication requirements
- HTTPS support status
- CORS compatibility
- Rate limiting information
- Pricing (all free)
- Example requests
- Response formats
- Related APIs in category

**Example**: `db/API/Weather/OpenWeather_API/readme.mdx`

### 2. API Categories (~45 entries)

**Path**: `db/Categories/{category-slug}/readme.mdx`

Each category includes:
- Category name and description
- List of APIs in category
- Common use cases
- Authentication patterns
- Popular APIs

### 3. API Authentication Methods (~5 types)

**Path**: `db/Auth_Methods/{method-slug}/readme.mdx`

Documentation for each auth type:
- Authentication flow
- How to obtain credentials
- Example implementation
- Security considerations
- APIs using this method

## API Access

### GitHub Repository

```bash
# Clone repository
git clone https://github.com/public-apis/public-apis.git

# JSON export available
curl https://api.publicapis.org/entries

# Filter by category
curl https://api.publicapis.org/entries?category=weather

# Filter by auth type
curl https://api.publicapis.org/entries?auth=apiKey

# Search by title
curl https://api.publicapis.org/entries?title=weather
```

### Rate Limits

- **GitHub API**: 60 requests/hour (unauthenticated)
- **publicapis.org API**: No official limits (community-run)

## Import Strategy

### Download Schedule

- **Daily sync**: Pull latest changes from GitHub
- **Webhook listener**: React to repository updates
- **Version tracking**: Monitor commit history

### ETL Pipeline

1. **Extract**: Parse README.md markdown table OR query publicapis.org API
2. **Transform**:
   - Parse markdown table rows
   - Extract API metadata
   - Generate slugs from API names
   - Categorize by domain
3. **Load**: Write MDX files to db/API/

### Incremental Updates

- Track last commit SHA
- Only process changed entries
- Preserve manual enhancements
- Maintain version history

## Usage Examples

### Find Weather APIs

```typescript
import { db } from '@mastra/core'

const weatherAPIs = await db.apis.findMany({
  where: { category: 'Weather' }
})
```

### Find No-Auth APIs

```typescript
const openAPIs = await db.apis.findMany({
  where: { auth: 'No Auth' }
})
```

### Search by HTTPS Support

```typescript
const secureAPIs = await db.apis.findMany({
  where: {
    https: true,
    cors: 'yes'
  }
})
```

### Find APIs by Description

```typescript
const cryptoAPIs = await db.apis.findMany({
  where: {
    description: { contains: 'cryptocurrency' }
  }
})
```
